hexo写作方式

方式一：

按hexo的方式，重新写：

$ hexo new [layout] <title>

按默认的配置，会在source/_post目录下生成新的.md文件，

编辑内容，格式如下：

---
title: ##标题
date: ##时间，格式为 YYYY-MM-DD HH:mm:ss
tag: ##文章关键词，多关键词格式为 keyword1,keywords2,...
categories: ##分类
keywords： ##文章关键词，多关键词格式为 keyword1,keywords2,...
description: ##文章描述
---
Markdown正文

说明：以 — 分隔的区域是Front-matter，用于指定个别文件的变量
Front-matter下面就是正文，可以写Markdown内容（注意：图片和链接要特殊处理）

方式二：

将已写好的.md文档，复制到source/_posts目录下，然后调整：

1.编辑.md文档，在顶层加入Front-matter：
如：
---
title: 标题
date: 2018-08-22 09:53:33
tag: [标签]
categories: [分类] 
---

插入图片

hexo对Markdown格式语法的支持

概述

Markdown部分语法在hexo中支持的不太友好

比如对图片资源的引用写法:

//Markdown写法：
![](/images/insert_image.jpg)

在hexo中不支持显示

解决办法：

  • 1.在Hexo 2时代，使用插件解决这个问题
  • 2.使用hexo的语法引入资源。——官网文档说明：随着Hexo 3发布，许多新的标签插件被加入到了核心代码中，
    使得你可以更简单地在文章中引用你的资源。
  • 3.使用图床+Markdown语法，引用图片(不需要安装插件)

第一种解决办法，既兼容老版本的hexo 2，对图片引用的语法上也不用做太多改动，改下图片相对路径就好

第二种新的语法引入资源，一个限制是版本>=hexo 3，也不太熟练

第三种方法也是比较科学的方法，推荐使用第一种和第三种


> ↓↓↓ 不管用哪种方法(除了第三种图床)，第一步都需要，启动hexo的资源文件夹功能，设置配置为true

hexo资源文件夹功能

对于那些想要更有规律地提供图片和其他资源以及想要将他们的资源分布在各个文章上的人来说，Hexo也提供了更组织化的方式来管理资源。这个稍微有些复杂但是管理资源非常方便的功能可以通过将 config.yml 文件中的 post_asset_folder 选项设为 true 来打开。

_config.yml
post_asset_folder: true //资源文件夹功能开关

当资源文件管理功能打开后，Hexo将会在你每一次通过 

hexo new [layout] <title>

命令创建新文章时自动创建一个文件夹。这个资源文件夹将会有与这个 markdown 文件一样的名字。将所有与你的文章有关的资源放在这个关联文件夹中之后，你可以通过相对路径来引用它们。

理解这段话的意思：

    1. 不仅仅只是自动帮你创建好一样名字的文件夹，方便你统一管理
    1. 要启用了这个开关，才可以通过相对路径引用到它们。——如果没有启用这个开关的话，是引用不到相对路径中的资源的。

遇到的坑

1.这里就遇到一个坑：
使用了新特性，将站点配置和主题部分配置放在同一个配置文件 source/_data/next.yml 中，

我起初觉得这个功能仅仅只是自动创建了一个文件夹，所以就只在根目录下的_config.yml中配置为true了，

我们启动和部署都是用参数–config …/next.yml 来使用自定义的配置的，这个next.yml中没有启用，结果是引用相对路径的图片怎么都看不到。

结论：
简单办法，在根目录下_config.yml和next.yml中都配置为true
——最终目的是：确保运行和发布时使用的配置中,资源文件夹功能开关是启用的

确保配置post_asset_folder: true
即可在运行(hexo s)，发布(hexo g)时，引用到相对路径的图片，从而正确生成插入图片
_posts
|- post1.md
|_ post1
    |- pic1.png

2.这里还有一个坑：

将已经写好的Markdown文档调整为hexo博客

手动调整的.md文件，新建的图片文件夹

    1. 必须和.md后缀前的名字一致
    1. 而且.md文件和图片文件夹必须都在source/_posts根目录

引入资源方法说明

使用Markdown语法在hexo中引入图片

解决方法具体步骤：

  • 1.确保配置为，post_asset_folder: true
  • 2.安装插件 hexo-asset-image：
      npm install hexo-asset-image --save
  • 3.建好放图片的文件夹，改引用图片的相对路径即可

    ===》（注意：新建文件夹与.md文件同名且在_posts）

使用hexo(标签插件)语法引入资源

hexo语法引入资源： 参考官网（标签插件/Image || Link）

Image

在文章中插入指定大小的图片。

{% img [class names] /path/to/image [width] [height] [title text [alt text]] %}

Link

在文章中插入链接，并自动给外部链接添加 target=”_blank” 属性。

{% link text url [external] [title] %}

例如：

{% img /hexo/writing/QQ亚索蜡笔小新.jpg [图片标题] %}

{% link 链接 https://sgven.github.io/ [链接悬浮标题] %}

引入图片需要注意的坑：

使用hexo 3 标签插件语法,引入图片需要注意的坑：

1.没有安装 hexo-asset-image 插件时

  • 1.1 引用路径只能定位到source根目录下，下面的配置启用无效：

      _config.yml
  post_asset_folder: true //资源文件夹功能开关

    ——如果安装了插件hexo-asset-image,则和Markdown语法一样，配置有效。

  • 1.2 经试验，并且在source根目录下 以下划线 _ 开头命名的文件夹，路径定位不到

  • 结论：所以在不用插件，仅使用hexo 3标签插件语法引用图片时，(或者配置不启用相对路径的功能)，此时只能引用到source目录下，且不是以 下划线 _ 开头的文件中的图片

    • 常用方法是：
      在source 目录下新建 image(img) 文件夹，将所有图片放入。

      ===》然后，引用图片时，使用类似下面的引用路径：

      /image/../xx.png

2.引用路径最左侧不能少了 / ，少了会报错。引用路径格式如下：

/image/../xx.png
