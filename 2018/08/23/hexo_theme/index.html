重点：修改主题配置，使用hexo新特性——Data Files

注意：这个特性是从hexo 3版本引起，需要 Hexo 的版本不低于 3

参考：特性 Benefits

  • 1.为什么要用这个特性？

    原因就是：

    • 我们要对主题做一些个性化处理，启用一些新功能等 都会修改主题配置文件。
    • 所以当我们从git上更新主题，git pull时就可能会冲突，从而导致升级主题失败。

  • 2.这个特性是干嘛的？

    • 通过这个特性，你可以将所有的主题配置放置在站点的 source/_data/next.yml 文件中。
    • 主题配置文件可以不用做任何修改。若后续版本有配置相关的改动时，你仅需在 next.yml 中做相应调整即可——这样就解决了升级和修改的冲突

  • 3.如何使用这个特性？

    • 1.请先确保你所使用的 Hexo 版本在 3 以上

    • 2.在站点的 source/_data 目录下新建 next.yml 文件（_data目录可能需要新建）

    • 3.迁移站点配置文件和主题配置文件中的配置到 next.yml 中

        站点配置文件：即根目录的_config.yml，全部复制进来，
  theme，记得还是指定你用到的主题，
  主题配置文件：也不用全迁移进来，迁移部分要用到的配置即可，做个性化配置处理。
  
  我认为，这里是theme:指定了引用的主题配置，
  然后next.yml中的所有配置会合并主题配置且覆盖键值重名的配置，
  这样就把需要改的主题配置复制到next.yml中，再个性化修改
  相当于assign(主题配置,next.yml)

    • 4.使用 –config source/_data/next.yml 参数启动服务器, 生成或者部署。

      例如:

    • hexo clean && hexo s –config source/_data/next.yml

    • hexo clean –config source/_data/next.yml && hexo g –config source/_data/next.yml

基础配置

1.Schemes，选择不同的外观

# Schemes
# scheme: Muse
# scheme: Mist
scheme: Pisces
# scheme: Gemini

2.语言

# 默认是英文的，设置为中文
language: zh-Hans

3.设置 菜单

1.(除首页home和归档archives菜单项)菜单项——要想显示有内容，
需要source对应目录下建有.md文件,用来生成显示的静态页面。
例如：
# 标签页tags： 添加一个标签页面，里面包含您网站中的所有标签
# 1.创建一个名为 tags 页面： 
# hexo new page "tags"
# 2.编辑标签页, 设置页面类型为tags：

    # title: All tags
    # date: 2014-12-22 12:39:04
    # type: "tags"
# 3.配置文件，去掉tags前的#，将tags添加到菜单中
    menu:
      home: /
      archives: /archives
      tags: /tags
    #   categories: /categories

同样的步骤,创建好分类页
# 分类页categories： 添加一个分类页面，里面包含您网站中的所有分类


2.配置
菜单前的 # 表示注释掉该行，即隐藏掉该条目

v5.1.2 文本和图标配置是分开的
# 设定菜单内容
# menu:
#   home: /
#   archives: /archives
#   about: /about
#   categories: /categories
#   tags: /tags

# 设定菜单项的图标
# menu_icons:
#     home: home
  #     about: user
  #     tags: tags
  #     categories: th
  #     archives: archive

v5.1.4 文本和图标的配置在一块
menu:
  home: / || home
  # about: /about/ || user
  # tags: /tags/ || tags
  categories: /categories/ || th
  archives: /archives/ || archive
  #schedule: /schedule/ || calendar
  #sitemap: /sitemap.xml || sitemap
  #commonweal: /404/ || heartbeat

若要修改菜单项的显示文本，以简体中文为例：
“需要修改” 简体中文对应的翻译文件 “languages/zh-Hans.yml”
（一般我们不修改,除非有必要，你想添加自定义的菜单选项，如menu下添加something）
# menu:
#   home: 首页
#   archives: 归档
#   categories: 分类
#   ...
#   something: 有料

5.设置「动画效果」

# 如果您比较在乎速度，可以将设置此字段的值为 false 来关闭动画。
use_motion: true  # 开启动画效果
# use_motion: false # 关闭动画效果

6.设置「背景动画」

# NexT 自带两种背景动画效果:canvas_nest 或 three_waves
# 注意： three_waves 在版本 5.1.1 中引入。只能同时开启一种背景动画效果。

# canvas_nest
# canvas_nest: true //开启动画
# canvas_nest: false //关闭动画

# three_waves
three_waves: true //开启动画
# three_waves: false //关闭动画

7.侧边栏社交链接

# social:
#     GitHub: https://github.com/your-user-name
#     微博: http://weibo.com/your-user-name
# social_icons:
#    GitHub: github
#   微博: weibo

数据统计与分析

站点统计：我们使用——不蒜子统计(此特性在版本 5.0.1 中引入,只需简单配置一下即可)

  • 1.参考next官网——数据统计与分析/不蒜子统计

    编辑 主题配置文件 中的 busuanzi_count 的配置项
    当enable: true时，代表开启全局开关。
    若site_uv、site_pv、page_pv的值均为false时，不蒜子仅作记录而不会在页面上显示。

  • 2.配置：

      # 算法a：pv的方式，单个用户连续点击n篇文章，记录n次访问量
  # 算法b：uv的方式，单个用户连续点击n篇文章，只记录1次访客数。
  # Show PV/UV of the website/page with busuanzi.
  # Get more information on http://ibruce.info/2015/04/04/busuanzi/
  busuanzi_count:
    # count values only if the other configs are false
    enable: true
    # custom uv span for the whole site
    site_uv: true
    site_uv_header: <i class="fa fa-user">本站访客数:</i>
    site_uv_footer:
    # custom pv span for the whole site
    site_pv: false
    site_pv_header: <i class="fa fa-eye">本站总访问量:</i>
    site_pv_footer:
    # custom pv span for one page only
    page_pv: false
    page_pv_header: <i class="fa fa-file-o"></i>
    page_pv_footer:

菜单添加——本地搜索服务Local Search

参考next官网——搜索服务/Local Search：

  • 1.安装 hexo-generator-searchdb，在站点的根目录下执行以下命令：

          $ npm install hexo-generator-searchdb --save

  • 2.配置：

      # 2. 编辑 站点配置文件，新增以下内容到任意位置：
  search:
    path: search.xml
    field: post
    format: html
    limit: 10000
  # 3. 编辑 主题配置文件，启用本地搜索功能：
  # Local search
  local_search:
    enable: true

添加字数统计、阅读时长

  • 1.安装 hexo-wordcount –save，在站点的根目录下执行以下命令：

      npm install hexo-wordcount --save

  • 2.编辑 主题配置文件，新增以下内容到任意位置：

      # Post wordcount display settings
  # Dependencies: https://github.com/willin/hexo-wordcount
  post_wordcount:
    item_text: true
    wordcount: true
    min2read: true
    totalcount: false
    separated_meta: true

解决相关问题，参考文章

  • 3.如果仅仅只是打开开关，部署之后会发现文章的【字数统计】和【阅读时长】后面没有对应的xxx字，xx分钟等字样，只有光秃秃的数字在那里。

解决方案：

​找到Blog\themes\next\layout_macro\post.swig 文件，将“字”、“分钟” 字样添加到如下位置即可：

<span title="{{ __('post.wordcount') }}">
     {{ wordcount(post.content) }} 字
</span>

    ...
<span title="{{ __('post.min2read') }}">
     {{ min2read(post.content) }} 分钟
</span>

然后可以看到显示如：阅读时长 ≈ 2 分钟 ,但比如不需要 ≈ 显示，修改这个地方：

{% if theme.post_wordcount.item_text %}
	        {{ __('post.min2read') }} ≈
	 {% endif %}

把这里的≈删除即可。（PS：这个是 html 的特殊字符的表示）
