## 撤消命令： 1.修复最新提交的便捷方式 $ git commit --amend - 用新的快照替换上一个提交 - 简单地编辑上一次提交的信息而不改变快照

2.取消暂存的文件

(use "git reset HEAD <file>..." to unstage)

NOTE 

加上 --hard 选项可以令 git reset 成为一个危险的命令
可能导致工作目录中所有当前进度丢失
不加选项地调用 git reset 并不危险 — 它只会修改暂存区域。

3.恢复已删除的文件 || 相当于还原revert，撤销对文件的修改，还原到上次提交的状态： 

git checkout --

例如

git checkout  ../yylib-react/apps/demos/modules/templates/CardFormTemplate.js
恢复则需要从新checkout： git checkout – <deleted_file>

4.撤销commit

reset命令， 主要的参数有soft,hard,mixed，它们告诉Git，当执行reset时，要对index和workingcopy做什么。

1.–soft 只回退了commit的信息，head指针改变，不会改变暂存区index和工作目录working copy的文件

git reset --soft：回退到某个版本，只回退了commit的信息，不会恢复到index file一级

2.–hard 彻底回退到某个版本，head,index,working copy全部回退

git reset --hard： a4cc94e(history看哈希码)
    彻底回退到某个版本，本地的源码也会变为上一个版本的内容

例如：

$ git reset 0985109 --hard
HEAD is now at 0985109 任务1750-->自动计算修正

3.–mixed是reset的默认参数，也就是当你不指定任何参数时的参数。它将重置HEAD到另外一个commit,并且重置index以便和HEAD相匹配，但是也到此为止。working copy不会被更改。

git添加和删除文件(目录)

git添加

git add (指定相对路径的)文件或目录——来添加文件或目录下所有文件(包括子目录)

git取消暂存

从暂存区域去除，文件变为未跟踪状态(Untracked)

  • 1.git reset 文件或目录——不管文件是否变更，都能直接从暂存区域移除，文件变为未跟踪状态(Untracked)
  • 2.git rm –cached 文件，目录需要加 -r——即

    git rm – cached -r 目录——如果文件变更，则失败：
      error: the following file has staged content 
  different from both the
  file and the HEAD:
      folder/inner/t1.txt
  (use -f to force removal)

==》所以，从暂存区域移除，使文件不被git跟踪，使用第一种命令更好：——git reset 文件或目录

git删除：

连同文件一起删除：

git rm -f 文件

git rm -f -r 目录

$ git rm -f -r folder

rm 'folder/inner/t2.txt'
rm 'folder/t1.txt'
rm 'folder/t2.txt'

## 与远程仓库交互的命令

  • 1.如果当前分支与多个主机存在追踪关系，则可以使用-u选项指定一个默认主机，这样后面就可以不加任何参数使用git push。

      $ git push -u origin master

NOTE

  • 不带任何参数的git push，默认只推送当前分支，这叫做simple方式，
  • 还有一种matching方式，会推送所有有对应的远程分支的本地分支，
  • Git 2.0之前默认使用matching，现在改为simple方式

在现有目录中(本地工程)初始化git仓库

如果你打算使用 Git 来对现有的项目进行管理，你只需要进入该项目目录并输入：

$ git init

该命令将创建一个名为 .git 的子目录，这个子目录含有你初始化的 Git 仓库中所有的必须文件，这些文件是 Git 仓库的骨干。 但是，在这个时候，我们仅仅是做了一个初始化的操作，你的项目里的文件还没有被跟踪。

$ git add .
$ git commit -m 'initial commit'

添加远程仓库

运行 git remote add (shortname) (url) 添加一个新的远程 Git 仓库

查看远程仓库

运行 git remote 命令， 它会列出你指定的每一个远程服务器的简写。

使用 git remote show [remote-name] 命令， 查看某一个远程仓库的更多信息。例如：

$ git remote show origin
