linux安装软件命令

一点常识先了解下

linux系统常用的有两种： CentOS 和 Ubuntu。

它们分别有各自的软件管理工具，

  • 一种是类似于windows的“程序和功能” rpm/dpkg，可以查看已安装的软件和卸载。

    • CentOS是： rpm
    • Ubuntu是： dpkg

  • 另一种是类似于软件管家的包管理工具 yum/apt-get，可以通过软件管家从软件仓库中获取软件。

    • CentOS是： yum
    • Ubuntu是： apt-get

Linux 允许我们配置从哪里下载软件，地点就在配置文件里面。
先配置远程仓库索引地址，再使用命令 “apt-get update” 更新仓库包索引。

使用 “apt-get update” 命令时，就会从“/etc/apt/sources.list”中配置的远程仓库检索，然后重新从源仓库中同步文件索引（即软件包索引）。

  • CentOS，配置文件在/etc/yum.repos.d/CentOS-Base.repo里。
  • Ubuntu，配置文件在/etc/apt/sources.list里。

如何在系统中启用一个仓库

例如，启用 epel-testing.repo

yum-config-manager --enable epel-testing

运行 yum repolist 命令来检查是否启用了 “epel-testing”。它被启用了，可以从列表中看到它。

如果你想同时启用多个仓库，使用以下格式。这个命令将启用 epel、epel-testing 和 ius 仓库：

yum-config-manager --enable epel epel-testing ius

如何在系统中禁用一个仓库

例如，禁用 epel-testing.repo 和 ius.repo

yum-config-manager --disable epel-testing ius

运行 yum repolist 命令检查 “epel-testing” 和 “ius” 仓库是否被禁用。被禁用了的那些仓库，则不能看到。

linux 防火墙命令

可以ping通虚拟机但无法访问8080端口

  • 虚拟机运行的linux系统，因为防火墙的原因，能ping通IP，但无法访问端口。需要用防火墙设置，对外开放你需要访问的端口。
  • 因为是默认安装的虚拟机所以没有开放80,8080等端口，打开端口即可

centos 6防火墙是iptables，centos 7防火墙是firewall-cmd

firewall-cmd

# 查看可访问的端口，显示没有
firewall-cmd --list-ports  

# 添加端口
# 成功的话，命令行均提示：“sucess”
firewall-cmd --zone=public --add-port=8080/tcp --permanent
firewall-cmd --zone=public --add-port=9090/tcp --permanent
firewall-cmd --zone=public --add-port=9091/tcp --permanent

# 重启防火墙 
firewall-cmd --reload

# 再次查询 
firewall-cmd --list-ports 显示可访问8080端口

# 关闭防火墙
systemctl stop firewalld.service

# 删除端口
firewall-cmd --zone=public --remove-port=8080/tcp --permanent

firewall-cmd: command not found

yum install firewalld

firewalld command not found

EPEL(Extra Packages for Enterprise Linux )即企业版Linux的扩展包，提供了很多可共Centos使用的组件，安装完这个以后基本常用的rpm都可以找到。

yum install epel-release

iptables

可能CentOS 6.10系统版本问题，依然找不到 firewalld 包，使用CentOS 6.10自带的 iptables 防火墙。

#防火墙基本操作
service iptables status            -- 防火墙服务是否启动
service iptables start            -- 启动防火墙
service iptables restart           -- 重启防火墙
service iptables  stop          -- 关闭防火墙

重启后永久性生效：
chkconfig iptables off             -- 永久关闭防火墙
chkconfig iptables on            -- 永久开启防火墙

1.确认、并启动防火墙

2.开启xx端口

vim /etc/sysconfig/iptables

加入以下内容

# 暴露8080端口
-A INPUT -m state --state NEW -m tcp -p tcp --dport 8080 -j ACCEPT
# 暴露9090端口
-A INPUT -m state --state NEW -m tcp -p tcp --dport 9090 -j ACCEPT

3.保存退出后重启防火墙

service iptables restart

//用列表的方式列出文件
ls -l
drwxr-xr-x 6 root root 4096 Oct 20 2017 apt

第一个字段：

- 第一个字符是**文件类型**
    - “-”，表示普通文件；
    - “d”，表示目录 
- 剩下的 9 个字符是**模式（即权限位）**

    - 3 个一组，每一组 rwx 表示“读（read）”“写（write）”“执行（execute）”；
    - 这三组分别表示文件所属的用户权限、文件所属的组权限以及其他用户的权限。

第二个字段： **硬链接（hard link）数目**

第三个字段： **所属用户**

第四个字段： **所属组**

第五个字段： 文件的大小

第六个字段： **文件被修改的日期**

最后： **文件名**

# 777 <=> 111,111,111 <=> rwxrwxrwx

linux软件管理

对于 Windows 系统，

  • 最方便的方式就是下载 .exe 安装文件，然后双击安装即可。
  • 第二种方式就是通过软件管家安装。
  • 第三种方式就是将安装好的压缩包，直接解压缩，再配置环境变量。

对于 Linux 系统，也提供了三种安装方式：

① 先下载再安装（使用软件包管理工具：rpm、dpkg进行安装）；

Linux也是类似的方法，你可以下载 .rpm 或者 .deb 文件，然后安装。这是Linux系统下的两种安装包，对应Linux的两大体系，CentOS、Ubuntu，前者使用rpm，后者使用deb。

第一种方式可以用来安装你自己下载的软件包

  • CentOS ：

      rpm -i file.rpm

  • （Debian）Ubuntu ：

      dpkg -i file.deb

rpm、dpkg其他命令

  • 查看安装的软件列表:

    linux下，使用如下命令：

      rpm -qa 
  dpkg -l（等价于dpkg --list）

    -q 就是 query，a 就是 all，-l 就是 list。

这就跟window下，通过控制面板查看一样：

  • 查看是否安装了某软件：

      rpm -qa |grep "关键词"
  dpkg -l |grep "关键词"

    |是管道，grep是搜索工具，将list结果通过|管道输入到grep中搜索，””可以省略，例如： dpkg -l |grep vim

② 通过软件管家（yum、apt-get）进行安装。

第二种安装方式，通过软件管家从软件仓库中获取软件。

  • CentOS 下使用 yum：

      //安装
  yum install java-11-openjdk.x86_64
  //卸载
  yum erase java-11-openjdk.x86_64

  • Ubuntu 下使用 apt-get：

      //安装
  apt-get install openjdk-8-jdk
  //卸载
  apt-get purge openjdk-8-jdk

Linux 允许我们配置从哪里下载软件，地点就在配置文件里面。
先配置远程仓库索引地址，再使用命令 “apt-get update” 更新仓库包索引。

使用 “apt-get update” 命令时，就会从“/etc/apt/sources.list”中配置的远程仓库检索，然后重新从源仓库中同步文件索引（即软件包索引）。

  • CentOS，配置文件在/etc/yum.repos.d/CentOS-Base.repo里。

  • Ubuntu，配置文件在/etc/apt/sources.list里。

③ 下载安装好的压缩包，直接解压缩（一切皆文件）

linux系统中，一切皆文件。

所以，无论是哪种方式进行安装，都是下载一些文件，然后将这些文件放在某个路径下，然后在相应的配置文件中配置一下。

这些文件在Linux中放的散一些：

  • 主执行文件会放在 /usr/bin 或者 /usr/sbin 下面；
  • 其他的库文件会放在 /var 下面；
  • 配置文件会放在 /etc 下面。

所以还有一种简单粗暴的方法：就是将安装好的压缩包路径直接下载下来，然后解压缩成为一个整的路径。

例如 JDK 的安装包：

  • windows有xx_bin.zip，这是 Windows 下常用的压缩模式；
  • Linux有_bin.tar.gz，这是 Linux 下常用的压缩模式。

如何下载压缩文件呢？

Linux 上面有一个工具 wget，后面加上链接，就能从网上下载了。

解压缩

下载下来后，我们就可以进行解压缩了。Linux 下面默认会有 tar 程序。如果是解压缩 zip 包，就需要另行安装。

yum install zip.x86_64 unzip.x86_64
apt-get install zip unzip

如果是 tar.gz 这种格式的，通过 tar xvzf jdk-XXX_linux-x64_bin**.tar.gz** 就可以解压缩了。

配置环境变量

对于 Windows 上 jdk 的安装，如果采取这种下载压缩包的格式，需要配置环境变量，设置JAVA_HOME和PATH。

在 Linux 也是一样的，通过 tar 解压缩之后，也需要配置环境变量，可以通过 export 命令来配置。

export JAVA_HOME=/root/jdk-XXX_linux-x64
export PATH=$JAVA_HOME/bin:$PATH

不过，export 命令仅在当前命令行的会话中管用。一旦退出重新登录，就不管用了。如何像 Windows 里面可以配置永远管用呢？

在当前用户的默认工作目录，例如 /root 或者 /home/cliu8 下面，有一个.bashrc 文件，这个文件是以点开头的，这个文件默认看不到，需要 ls -la 才能看到，a 就是 all。每次登录的时候，这个文件都会运行，因而把它放在这里。这样登录进来就会自动执行。当然也可以通过 source .bashrc 手动执行。

要编辑.bashrc 文件，可以使用文本编辑器 vi，也可以使用更加友好的 vim。需要另行安装。

yum install vim
apt-get install vim

运行程序

Windows 下的程序，如果后缀名是 exe，双击就可以运行了。

linux程序执行条件

Linux 不是根据后缀名来执行的。它的执行条件是这样的：只要文件有 x 执行权限，都能到文件所在的目录下，通过 ./filename 运行这个程序。（如果放在 PATH 里设置的路径下面，就不用 ./ 了，直接输入文件名就可以运行了，Linux 会帮你找。）

① 在交互命令行里面运行

这是Linux 执行程序最常用的一种方式，通过 shell 在交互命令行里面运行。

这样执行的程序可能需要和用户进行交互，这种方式比较适合运行一些简单的命令，缺点是，一旦当前的交互命令行退出，程序就停止运行了。

② 后台运行

于是，就有了 Linux 运行程序的第二种方式，后台运行。

  • 这个时候，我们往往使用 nohup 命令。意思是 no hang up（不挂起），也就是说，当前交互命令行退出的时候，程序还要在。

  • 当然这个时候，程序不能霸占交互命令行，而是应该在后台运行。最后加一个 & ，就表示后台运行。

  • 另外一个要处理的就是输出，原来什么都打印在交互命令行里，现在在后台运行了，输出到哪里呢？输出到文件是最好的。

最终命令的一般形式为：nohup command >out.file 2>&1 &

“1”表示文件描述符 1，表示标准输出，“2”表示文件描述符 2，意思是标准错误输出，
“2>&1”表示标准输出和错误输出合并了。合并到哪里去呢？到 out.file 里。

进程如何关闭呢？

假设启动的程序包含某个关键字，那就可以使用下面的命令。

ps -ef |grep 关键字  |awk '{print $2}'|xargs kill -9

其中 ps -ef 可以单独执行，列出所有正在运行的程序，grep 通过关键字找到启动的程序。

awk 工具可以很灵活地对文本进行处理，这里的 awk ‘{print $2}’是指第二列的内容，是运行的程序 ID。我们可以通过 xargs 传递给 kill -9，也就是发给这个运行的程序一个信号，让它关闭。如果你已经知道运行的程序 ID，可以直接使用 kill 关闭运行的程序。

③ 以服务的方式运行

在 Windows 里面还有一种程序，称为服务。这是系统启动的时候就在的，我们可以通过控制面板的服务管理启动和关闭它。

Linux 也有相应的服务，这就是Linux程序运行的第三种方式，以服务的方式运行。

例如常用的数据库 MySQL，就可以使用这种方式运行。

  • 在 Ubuntu 中，安装 MySQL

      apt-get install mysql-server

  • 启动 MySQL

      systemctl start mysql

  • 设置开机启动

      systemctl enable mysql

之所以成为服务并且能够开机启动，是因为在 /lib/systemd/system 目录下会创建一个 XXX.service 的配置文件，里面定义了如何启动、如何关闭。

在 CentOS 里有些特殊，MySQL 被 Oracle 收购后，因为担心授权问题，改为使用 MariaDB，它是 MySQL 的一个分支。

  • 在 CentOS 中，安装MySQL

      yum install mariadb-server mariadb

  • 启动MySQL

      systemctl start mariadb

  • 设置开机启动

      systemctl enable mariadb

同理，会在 /usr/lib/systemd/system 目录下，创建一个 XXX.service 的配置文件，从而成为一个服务。

多种linux系统的对比

linux系统有很多种，比较常用的是CentOS、Debian、Ubuntu，它们都是开源免费。其中，Ubuntu 是从Debian分支出来的。Alpine是一个轻型的Linux，docker镜像用它作为基础镜像，则会非常小。

CentOS VS Ubuntu

观点1： CentOS适用于服务器，Ubuntu则适用于个人桌面

  • CentOS：适用于服务器

    作为 redhat 社区版本，其稳定性毋庸置疑。在国内的linux 运维环境中， centos 是占大头的。很多公司都是用CentOS来代替商业版的Red Hat Linux。

  • Ubuntu：适用于个人桌面使用

    Ubuntu的开发目的是为了使个人电脑变得简单易用，Ubuntu 桌面版提供了友好的安装界面，虽然也提供相应的企业服务器，不过专业的还是CentOS。

观点2： CentOS和Ubuntu平分秋色

我们常用linux系统来部署生产环境，所以这里还是比较CentOS和Ubuntu服务器的优劣。

  • 其实，在国际上，采用 centos 和采用 ubuntu 作为服务端是其实是五五开的，两者各有特色；
  • 最最重要的差异就是，ubuntu 相比 centos 在更新策略上要积极一些。在 centos 上如果不配置扩展源的话，很多软件都是没办法yum安装的，比如 nginx。而 ubuntu 系统，就没有这个烦恼啦，直接 apt install nginx 就可以咯！

总结就是，用于服务器上的话，如果你希望更加容易的获得较新版本的软件，使用 ubuntu 系统可能会更加方便一些。当然，如果保守一些，希望稳定一些，还是选择 centos 靠谱。

Alpine Linux

Small. Simple. Secure.

Alpine Linux is a security-oriented, lightweight Linux distribution based on musl libc and busybox

Alpine Linux是一个面向安全的轻型的Linux发行版。

Alpine Linux采用了 musl libc 和 busybox以减小系统的体积和运行时资源消耗。

在保持瘦身的同时，Alpine Linux还提供了自己的包管理工具apk。

关键的是，相比于其他Linux的Docker镜像，它的容量非常小，仅仅只有5MB。

例如，elasticsearch-head的docker镜像，5-alpine就是用Alpine Linux作为基础镜像

docker pull mobz/elasticsearch-head:5-alpine

docker pull mobz/elasticsearch-head:5

Linux 上手难，学习曲线陡峭，它的学习过程更像一个爬坡模式。这些坡看起来都很陡，但是一旦爬上一阶，就会一马平川。

linux入门难，我也吃过这种苦，所以，我把学习linux操作系统的经验总结了下，希望能帮助到你入门。

linux要成为大牛，大概要爬六个坡，不过对我们一般人来说，爬上第一个坡，“熟悉linux命令” + “实践”就够用了，深入的话，可以了解一些感兴趣的linux内核机制，比如CFS调度策略。

先来看看第一个坡：

上手 Linux 的第一步，要先从 Windows 的思维习惯，切换成 Linux 的“命令行 + 文件”使用模式。

爬第一个坡的阶段，就是要熟练使用linux命令行，熟练大概就是你对要用什么命令心里有数，命令的意思你能搞懂。基本的常用命令你都会用，对于有的命令最好是掌握深入一点，比如“ | grep”是什么意思，修改文件权限的时候“chmod 777 file”，chmod后面的数字是什么意思。

有可能连linux环境都没有，所以，会需要自己搭建一个linux操作系统。多是在本地windows系统下安装一个VMware虚拟机管理软件，在VMware中搭建一个linux系统（虚拟机），操作系统弄好后还会需要安装一些你需要的软件。这个过程中间会遇到很多问题，这些问题大多都可以去网上找到解决办法。这个过程我后面也有记录，可以参考借鉴。

在搭建linux系统环境之前，可以先对linux多了解了解，比如几大常用的linux系统CentOS、Ubuntu有何区别，常用的命令，常用的工具软件等。再搭建linux，然后再多操作、熟练使用linux命令。

这个过程可能非常痛苦，也不必急于求成一口吃一个胖子，慢慢来，只要你坚持下去，你就会爬上第一个坡了。
