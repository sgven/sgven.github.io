springBoot入门指南

官网指南

需要什么

  • JDK 1.8 or later
  • Gradle 4+ or Maven 3.2+
  • you can also import demo code into your IDE:
    • IntelliJ IDEA

创建SpringBoot工程 or 下载(demo)：

  • 可以在官网https://start.spring.io/ 生成工程(可在界面中自行选择依赖)
  • or 在你的workspace目录下 通过git clone下载demo
      git clone https://github.com/spring-guides/gs-spring-boot.git

构建(使用Maven，构建demo)：

在IDEA中，右键pom.xml，Add as Maven Project。等待下载好maven依赖。

运行应用程序(使用Maven，运行demo)：

使用Maven、Jar运行SpringBoot

mvn package && java -jar target / gs-spring-boot-0.1.0.jar

注意：

  • 1.如果碰到错误: 找不到或无法加载主类 com.example.demo.DemoApplication，需要在第一次运行SpringBoot时，执行上面的命令，来启动SpringBoot：

  • 2.还需要注意，如果代码做了修改，需要再次执行上面的命令进行编译，不然改动不会生效。(不注意这个的话，可能你会一时摸不着头脑，通过@SpringBootApplication的类的main方法启动，程序能正常跑起来，怎么改动就是没生效呢)

使用Groovy、SpringBoot’s CLI运行SpringBoot

Spring Boot还支持Groovy，允许您使用一个文件构建Spring MVC Web应用程序

官网：

Spring Boot also has Groovy support, allowing you to build Spring MVC web apps with as little as a single file.

  • 1.创建一个名为app.groovy的新文件，并在其中添加以下代码：

      @RestController
  class ThisWillActuallyRun {
  
      @RequestMapping("/")
      String home() {
          return "Hello World!"
      }
  
  }

    文件的位置无关紧要。

  • 2.安装Spring Boot的CLI

    • 给本地Maven仓库下载groovy依赖，pom.xml中添加如下依赖：

        <!-- groovy -->
  <dependency>
      <groupId>org.codehaus.groovy</groupId>
      <artifactId>groovy-all</artifactId>
      <version>2.4.15</version>
  </dependency>

    • 运行如下：

        spring run app.groovy

    • 进入：http://localhost:8080/，显示：

        Hello World!

通过maven命令运行SpringBoot

因为POM使用了spring-boot-starter-parent，所以可以用 run 来启动应用程序。从项目根目录中键入 mvn spring-boot:run 以启动应用程序

mvn spring-boot:run


从无到有，通过pom.xml，创建SpringBoot工程

创建pom

<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>

    <groupId>com.example</groupId>
    <artifactId>myproject</artifactId>
    <version>0.0.1-SNAPSHOT</version>

    <parent>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-parent</artifactId>
        <version>2.0.5.RELEASE</version>
    </parent>

    <!-- Additional lines to be added here... -->

</project>

继承Starter Parent

spring-boot-starter-parent主要作用就是统一管理依赖版本(不实际引入依赖)，引入其他starter不需要指定版本了，避免版本冲突。


可以看到starter-parent的pom中继承自spring-boot-dependencies，然后spring-boot-dependencies中的dependencyManagement维护了所有spring-boot需要的依赖版本。

<!-- 只需要在此依赖项上指定Spring Boot版本号。如果导入其他启动器，则可以安全地省略版本号 -->
<parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>2.0.5.RELEASE</version>
</parent>

<!-- 通过如下设置，来覆盖单个依赖项 -->
<properties> 
    <spring-data- releasetrain.version>
 Fowler-SR2 </spring-data-releasetrain.version> </ properties>

不继承starter-parent

您可能拥有自己需要使用的公司标准父级，或者您可能更愿意明确声明所有Maven配置。

如果您不想使用 spring-boot-starter-parent，您仍然可以通过使用 scope=import 依赖项来保持依赖项管理的好处，如下所示：

为什么用scope=import，请参考maven依赖版本统一管理。

<dependencyManagement>
        <dependencies>
        <dependency>
            <!-- Import dependency management from Spring Boot -->
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-dependencies</artifactId>
            <version>2.0.5.RELEASE</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>

创建可执行的Jar

Java没有提供加载嵌套jar文件的标准方法（jar文件本身包含在jar中）

注意：要创建可执行jar，需要添加 spring-boot-maven-plugin 到我们的 pom.xml

<build> 
    <plugins> 
        <plugin> 
            <groupId> org.springframework.boot </ groupId> 
            <artifactId> spring-boot-maven-plugin </ artifactId> 
        </ plugin> 
    </ plugins> 
</ build>

编写代码

By default，Maven compiles sources from src/main/java，So you need to create a Java file. 最简单的一个示例，src/main/java/Example.java：

import org.springframework.boot.*;
import org.springframework.boot.autoconfigure.*;
import org.springframework.web.bind.annotation.*;

@RestController
@EnableAutoConfiguration
public class Example {

    @RequestMapping("/")
    String home() {
        return "Hello World!";
    }

    public static void main(String[] args) throws Exception {
        SpringApplication.run(Example.class, args);
    }

}

SpringBoot项目结构

SpringBoot项目结构的最佳实践：主应用程序类放在其他类之上的根包中(主类or启动类一般使用@SpringBootApplication注解) 。如下：

com 
 +  -  example 
     +  -  myapplication 
         +  -  Application.java 
         | 
         +  - 客户
         | +  -  Customer.java 
         | +  -  CustomerController.java 
         | +  -  CustomerService.java 
         | +  -  CustomerRepository.java 
         | 
         +  - 订单
             +  -  Order.java 
             +  -  OrderController.java 
             +  -  OrderService.java 
             +  -  OrderRepository.java

根包

根包就是放在其他类之上的包。

Java推荐的包命名约定并使用反向域名（例如，com.example.project通常就是根包）

默认包

当一个类不包含package声明时，它被认为是在“默认包”中。

  • 通常不鼓励使用“默认包”，应该避免使用。

  • 这可能会导致使用了 @ComponentScan, @EntityScan, or @SpringBootApplication注解的SpringBoot应用程序的特殊问题。

    例如：

      ** WARNING ** :Your ApplicationContext is unlikely to start due to a @ComponentScan of the default package.

  due to org/springframework/dao/DataAccessException not found. Make sure your own configuration does not rely on that class. This can also happen if you are @ComponentScanning a springframework package (e.g. if you put a @ComponentScan in the default package by mistake)

    可以通过basePackageClasses等属性来细粒度的定制@ComponentScan自动扫描的范围，不指定则默认从声明@ComponentScan所在类的package进行扫描。因为启动类在默认包下，所以默认包下只有启动类DemoApplication加入到Spring的上下文Context中，这样SpringBoot运行还需要的其他的类就找不到了。

    SpringApplication.run(DemoApplication.class, args);

  • 解决办法：

    • 1.显示指定要扫描的包

      @ComponentScan(basePackageClasses＝{“包名1”,”包名2”})
      或者
      @SpringBootApplication(scanBasePackages = {“包名1”,”包名2”})

    • 2.按SpringBoot的建议，启动类放在根包上。

如果想在其他包下引用默认包中的类：
比如因为一些历史原因，有的类是放在默认包中，这时其他包中想引用它们。

  • Java其它包中无法引用缺省包(default package)，因为其它的包如果要调用缺省包中的类,必须插入包名.但是缺省包却无名,所以不能够直接调用。

    就必须通过反射或其他一些间接方法访问默认包中的类

@SpringBootApplication注解

@SpringBootApplication包含下面三个注解：

  • @EnableAutoConfiguration：启用Spring Boot的自动配置机制
  • @ComponentScan：@Component在应用程序所在的包上启用扫描（请参阅最佳实践）
  • @Configuration：允许在上下文中注册额外的bean或导入其他配置类

可以组合其中单个的注解。如下，没有用@ComponentScan来自动检测@Component注释的类，而是通过@Import显式导入用户定义的bean：

@Configuration
@EnableAutoConfiguration
@Import({ MyConfig.class, MyAnotherConfig.class })
public class Application {

    public static void main(String[] args) {
            SpringApplication.run(Application.class, args);
    }

}
