springBoot特性

##Maven依赖：

devtools自动重启

<!-- 类似JRebel的热插拔 替换有变化的字节码
    optional true表示可选依赖，防止devtools传递到其他模块 -->
<dependencies>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-devtools</artifactId>
        <optional>true</optional>
    </dependency>
</dependencies>

spring-boot-devtools只要类路径上的文件发生更改，就会自动重新启动应用程序。

触发重启

DevTools监视类路径资源，因此触发重新启动的唯一方法是更新类路径。
在Eclipse中，保存修改后的文件会导致更新类路径并触发重新启动。
在IntelliJ IDEA中，构建项目（Build -> Build Project）具有相同的效果。

configuration-processor配置文件处理器

<!-- 配置文件处理器，在application.properties编写or输入. 就会有提示 -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-configuration-processor</artifactId>
    <optional>true</optional>
</dependency>

Spring Beans和依赖注入

@ComponentScan

扫描@Component, @Service, @Repository, @Controller等组件，自动注册为Spring中的Beans。

@Autowired

Spring 2.5 引入了 @Autowired 注释，用来注入Spring中的Beans，完成自动装配的工作。

@Autowired的原理

在启动spring IoC时，容器自动装载了一个AutowiredAnnotationBeanPostProcessor后置处理器。当容器扫描到@Autowied、@Resource或@Inject时，就会在IoC容器自动查找需要的bean，并装配给该对象的属性。

注意：在使用@Autowired时，首先在容器中查询对应类型的bean：

  • 如果查询结果刚好为一个，就将该bean装配给@Autowired指定的数据
  • 如果查询的结果不止一个，那么@Autowired会根据名称来查找。
  • 如果查询的结果为空，那么会抛出异常。解决方法时，使用required=false

Using YAML Instead of Properties

Springboot的出现目的也是在一定程度上去掉XML的繁琐配置，在Springboot中，推荐使用properties或者YAML文件来完成配置，对于较复杂的数据结构，YAML又远远优于properties。所以更推荐使用YAML。——>

YAML语法快速入门。

加载YAML

Spring Framework提供了两个方便的类，可用于加载YAML文档。该 YamlPropertiesFactoryBean 负载YAML作为Properties和 YamlMapFactoryBean 负载YAML作为Map。

  • demo1:

    YAML文档：

      environments:
      dev:
          url: http://dev.example.com
          name: Developer Setup
      prod:
          url: http://another.example.com
          name: My Cool App

    前面的示例将转换为以下Properties：

      environments.dev.url=http://dev.example.com
  environments.dev.name=Developer Setup
  environments.prod.url=http://another.example.com
  environments.prod.name=My Cool App

  • demo2:

    YAML文档：

      my:
  servers:
      - dev.example.com
      - another.example.com

    Properties：

      my.servers[0]=dev.example.com
  my.servers[1]=another.example.com

    @ConfigurationProperties：

    可以通过Spring Boot的@ConfigurationProperties 绑定到这个Properties

      @ConfigurationProperties(prefix="my")
  public class Config {

      private List<String> servers = new ArrayList<String>();
  
      public List<String> getServers() {
          return this.servers;
      }
  }

YAML缺点

使用@PropertySource注释无法加载YAML文件。如果需要以这种方式加载配置，则需要使用Properties文件。

@PropertySource

作用

加载外部资源文件，注意：不能少了** classpath: **

@PropertySource(value = "classpath:application.properties")
// 这么写是错的
@PropertySource(value = "application.properties")
会报java.io.FileNotFoundException: Could not open ServletContext resource [/application.properties]

改为前面有classpath:的，如@PropertySource(value = “classpath:application.properties”)，mvn package编译打包成功：

@Configuration

基于CGlib代理实现

作用

声明当前类是一个配置类，相当于一个Spring的XML配置文件，与@Bean配合使用。

@SpringBootConfiguration

@SpringBootConfiguration上包含了一个@Configuration注解，是来声明当前类是SpringBoot应用的配置类。

EnableAutoConfiguration

这个注释告诉告诉SpringBoot基于你所添加的jar依赖关系，去“猜测”你想要如何配置Spring。比如dependency中引入了spring-boot-starter-web，
因为这个启动器中添加了tomcat、SpringMVC的依赖，
此时自动配置就知道你是要开发一个web应用，所以就帮你完成了web及SpringMVC的默认配置了！

@Import

把某个类的实例Bean注入到spring的IOC容器中，跟通过@ComponentScan扫描自动注册不同，是手动显示注入的。

@Enable**Configuration

@EnableAutoConfiguration原理及自定义扩展

Spring Boot很好地利用了Spring 4框架的功能，可以创建一个自动配置的可执行JAR。 

@ConditionalOnBean（仅仅在当前上下文中存在某个对象时，才会实例化一个Bean）
@ConditionalOnClass（某个class位于类路径上，才会实例化一个Bean）
@ConditionalOnExpression（当表达式为true的时候，才会实例化一个Bean）
@ConditionalOnMissingBean（仅仅在当前上下文中不存在某个对象时，才会实例化一个Bean）
@ConditionalOnMissingClass（某个class类路径上不存在的时候，才会实例化一个Bean）
@ConditionalOnNotWebApplication（不是web应用）
